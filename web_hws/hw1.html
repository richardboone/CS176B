CS176B - HOMEWORK ASSIGNMENT #1
Due: January 27, 2019 (By 11:59pm)

Objectives

    This homework is designed to have you explore some of the tools we discussed in class to better understand the network to which you are connected.

Assignment Details

    All of these tools should be available on any machine running a UNIX-based operating system, i.e. the machines in CSIL. Many are also available on non-UNIX machines, but the syntax may vary. You will want to make extensive use of the man pages for each command (either via the command line or via the Internet).

    A. ifconfig or ipconfig

    1. Determine the network interface configuration of the machine you are using (e.g., "ifconfig -a" or "ipconfig /all"). Capture the output (screenshot) and include it as part of your answer. Identify and explain as much of what is printed as you can.
    2. Using the "ip" command in Unix, what command would you enter to get similarly detailed information? For example, "ip xxxxx"

    B. ARP

    1. Run "arp -a" on your machine. Capture the output (screenshot) and include it as part of your answer. Explain each part of what is printed.

    2. If you try to use the arp command to add or delete an entry to the ARP table, what happens? Why do you suppose this the case?

    3. There is another indirect way of affecting the contents of the ARP table. How can you affect (either add, delete, or change) entries in the ARP table? Use this mechanism to add at least two new hosts to the ARP table and capture the output (screenshot) of the new ARP table and include it as part of your answer.

    4. How long do entries stay cached in the ARP table? Describe a trail-and-error method to discover the timeout value.

    5. How is the output of "arp -a" different from "ip neighbor"?

    6. What will happen if two different IP addresses assigned to each of two different hosts both map to the same Ethernet address? Be specific on how hosts on the subnet operate and respond.

    C. traceroute

    1. Explain in detail how traceroute works.

    2. Perform a traceroute from your machine to jnto.org.au. Include a copy of the output and explain what happened including a description of what each of the fields means.

    3. Perform some other traceroutes. Include a copy of the traceroute that completed successfully and had the most number of hops.

    4. Here's the challenge for this question: we want to traceroute to a non-existent machine outside of UCSB. The first problem is how do we KNOW that a particular machine does not exist? If we just pick an IP address and ping it, why is that not sufficient?

    5. Determine an IP address for a machine that you know definitely does not exist (but is still a valid public IP address in the Class A, B, or C range). Explain how you determined that the IP address you chose is not being used. Do a traceroute to that machine. Include a copy of the result.

    6. Identify a completely different (for all four parts of the IP address) IP address for a non-existent machine. Do a traceroute to this machine. Include a copy of this result.

    7. Repeat the above exercise (but you don't have to include traceroutes to more than two machines in your writeup) enough times so you can detect a pattern about what is happening. What pattern exists and why?

    8. What is tracepath and how is it different than traceroute?

    D. Router Looking Glass

    1. Use "http://alice.ja.net" to show the route from this router (from the drop-down menu) to UCSB (resolve the domain and provide its IP as the argument). Explain as much of the output as possible--use Google to learn about the information displayed.

    2. Use "http://alice.ja.net" to perform a traceroute to 128.111.1.1. Include a copy of the output.

    E. netstat (or ss)

    1. What is netstat (or ss) and what is it used for?

    2. What parameters for netstat (or ss) should you use to show all the TCP connections established? Include a printout of this list for your machine. Be sure to explain what all fields are.

    3. What option of netstat (or ss) can be used to display network interface status? By using netstat (or ss), figure out the number of interfaces on your machine. In the output you will find an interface named "lo0" as a loopback interface. What is this interface, and what are some of the ways it is useful?

    F. nslookup (or dig)

    1. What is the IP address for the machine cs.princeton.edu?

    2. What local DNS server is this information coming from? How does your computer know to use this DNS server?

    3. Here is the problem: I want to find the IP address of where my email to "somebody@hotmail.com" goes. What you really need to do is find the mail exchanger for hotmail.com. There is an option in nslookup (or dig) that tells you what the mail exchanger is for hotmail.com. Figure out the exact syntax of the format of this command, and execute it. Now what is the IP address of where my email to hotmail.com goes?

    G. whois

    1. Who is the administrative contact for the ucsb.edu network?

    2. The host "morticia.cc.gatech.edu" is not in the whois database, but its IP address is. What is the address of the organization that manages this machine?

    3. If the host "wave3.mit.edu" is being used to attack my machine. Who should I call and what is their phone number?

    4. Like the hotmail example, I have another friend I send email to at gmail. Her email address is "something@gmail.com" What can you say about the coordinator for the gmail network, i.e. contact info, etc? Include information about the steps you took and the results of the queries you made to figure out this information.

    5. What company owns IP address 198.182.196.56? What machines act as the DNS nameserver for the domain that 198.182.196.56 is in?

    H. Misc

    1. Do a DNS lookup for whitehouse.gov? What IP address is returned? If you enter that IP address into the browser bar of a browser, what happens?

    2. If you enter "whitehouse.gov" into the browser bar of a browser, what happens? Explain any differences between what happened in the previous question and for this question.

    3. To whom is the appropriate person to send abuse complaints related to whitehouse.gov's web site? Comment on the domain of this person's IP address.

    4. To whom is the appropriate person to send abuse complaints related to whitehouse.gov's email system? Comment on the domain of this person's IP address.
